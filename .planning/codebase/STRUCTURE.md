## Directory Purposes

**`references/`:**
- Purpose: Stores the curated audit rule definitions -- the core intellectual property of the tool
- Contains: 5 markdown files, one per audit category (AEO, GEO, SEO Technical, SEO On-Page, Structured Data)
- Key files: All 5 are equally important. Each follows the same format: Source section, then CRITICAL/IMPORTANT/NICE TO HAVE check tiers
- Update frequency: Manually, roughly quarterly, when source guidelines change

**`scripts/`:**
- Purpose: Shell scripts that Claude Code calls during audits
- Contains: Currently one script (`pagespeed.sh`)
- Key files: `scripts/pagespeed.sh` -- calls Google PageSpeed Insights API, returns JSON with Lighthouse scores and CrUX Core Web Vitals

**`docs/` (gitignored):**
- Purpose: All generated output from audit runs and planning sessions
- Contains: Three subdirectories for different output types
- Key files: `docs/w-audit/audit-{domain}-{date}.md` (audit reports), `docs/logs/audit-log-{domain}-{date}.md` (structured audit logs)
- Note: Entire directory is gitignored -- reports are local-only

**`.claude/`:**
- Purpose: Claude Code project-level configuration
- Contains: `settings.json` for hook definitions; `hooks/` for hook scripts
- Key files: `settings.json` defines PreToolUse and Stop hooks
- Note: `hooks.json`, `hooks/`, and `session-logs/` are gitignored

**`.playwright-mcp/`:**
- Purpose: Auto-generated Playwright MCP server working directory
- Contains: Browser session data, screenshots
- Generated: Yes
- Committed: No (should be gitignored, currently not explicitly listed)

**`.planning/codebase/`:**
- Purpose: GSD system codebase analysis documents
- Contains: Architecture, structure, conventions, and concerns docs
- Generated: By GSD mapping commands
- Committed: Yes

## Naming Conventions

**Files:**
- Reference files: lowercase with hyphens, match category slug (`aeo.md`, `seo-technical.md`, `structured-data.md`)
- Scripts: lowercase with hyphens (`pagespeed.sh`)
- Audit reports: `audit-{domain}-{YYYY-MM-DD}.md`
- Comparison reports: `compare-{domain1}-vs-{domain2}-{YYYY-MM-DD}.md`
- Audit logs: `audit-log-{domain}-{YYYY-MM-DD}.md`
- Root project files: UPPER-CASE with hyphens (`SKILL.md`, `PROJECT-BRIEF.md`, `README.md`)

**Directories:**
- Lowercase, short names (`references/`, `scripts/`, `docs/`)
- Nested output dirs use short slugs (`w-audit/`, `logs/`, `plans/`)

## Where to Add New Code

**New Audit Category:**
1. Create a new reference file at `references/{category-slug}.md` following the existing format (Source section, then CRITICAL/IMPORTANT/NICE TO HAVE tiers with checkbox items)
2. Update `SKILL.md` to include the new category in: the reference file loading list (line 181-186), the scoring weights (line 207-211), and the report template sections (line 248-277)
3. Add the category weight to the scoring formula -- weights must sum to 100%

**New Helper Script:**
- Place in `scripts/` with a descriptive hyphenated name
- Include a usage comment at top, check for required env vars, exit with error message if missing
- Follow the pattern in `scripts/pagespeed.sh`

**New Reference/Rule Updates:**
- Edit the relevant file in `references/`
- Update the "Last reviewed" date in the Source section
- Follow the existing check format: `- [ ] {Description} -- {evidence/citation}`
- Mark conditional checks explicitly with `**CONDITIONAL: {condition}**`

**New Output Type:**
- Create a new subdirectory under `docs/` (e.g., `docs/new-output/`)
- Add the naming pattern to `SKILL.md`
- The `docs/` directory is gitignored, so no git changes needed

**New Claude Code Hook:**
- Add hook script to `.claude/hooks/`
- Register it in `.claude/settings.json` under the appropriate event (PreToolUse, PostToolUse, Stop, etc.)
- Hook scripts are gitignored
