Create the scoring test infrastructure: a standalone scoring script, synthetic test fixtures, and a golden-file test runner.

Purpose: Scoring regressions happen silently when reference files change (adding a CRITICAL check increases the denominator by 3, dropping all scores). <!-- Scoring formula from SKILL.md Section 4 (lines 198-221): -->

Scoring rules:
- Critical checks: 3 points each
- Important checks: 2 points each
- Nice to Have checks: 1 point each
- PASS = full points
- WARNING = half points (rounded down via floor)
- FAIL = 0 points
- N/A = excluded from numerator AND denominator
- UNTESTABLE = excluded from numerator AND denominator
- Score = (points earned / points possible) * 100
- If possible == 0: score = 100 (no testable checks = perfect)

Weight multipliers (for overall grade, NOT used in per-category scoring):
- AEO: 25%, GEO: 25%, SEO Technical: 20%, SEO On-Page: 15%, Structured Data: 15%

Letter grades: A+ (95+), A (90+), A- (85+), B+ (80+), B (75+), B- (70+), C+ (65+), C (60+), C- (55+), D (50+), F (<50)

<!-- Research-provided scoring implementation: -->
```python
WEIGHTS = {"critical": 3, "important": 2, "nice_to_have": 1}
RESULT_MULTIPLIERS = {"PASS": 1.0, "WARNING": 0.5, "FAIL": 0.0}

def score_checks(checks):
    earned = 0
    possible = 0
    for check in checks:
        if check["result"] in ("N/A", "UNTESTABLE"):
            continue
        weight = WEIGHTS[check["severity"]]
        possible += weight
        multiplier = RESULT_MULTIPLIERS[check["result"]]
        earned += int(weight * multiplier)  # floor for WARNING
    if possible == 0:
        return 100
    return int(earned / possible * 100)
```
